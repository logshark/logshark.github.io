What is uvm_root?

uvm_root 是一個單例類別，在驗證環境中會有個以uvm_rootclass 產生的instance是uvm_top是所有 UVM component最top level的容器(container)，它會在UVM初始化的時候自動創建，並在整個simulation過程中存在。使用者不能在create其他uvm_root instance。

Implicit Top-Level

uvm_top是隱含的最上層的component，只要沒有指定parent component的uvm component object就會變成 uvm_top 的子元件。
例如上面範例的m_env_3和m_env_4沒有指定parent，那他們的parent就是uvm_top。

Search Functionality

uvm_top instance can be used to search for components by their full hierarchical name in a testbench using wildcards. The component hierarchy will be searched in top to bottom fashion in alphanumeric order. The functions find() and findall() can be used to return component handles that match the provided wildcard pattern.

uvm_top instance可用於使用通配符在測試台中按元件的完整層次名稱搜尋元件。將以字母數字順序從上到下的方式搜尋元件層次結構。函數 find() 和 findall() 可用來傳回與所提供的通配符模式相符的元件句柄。

uvm_top instance可以搜尋(find()/find_all())和印出整個階層結構(print_topology())的function可以使用，上面的範例有使用find()/find_all()搜尋component，也有使用print_topology()印出整個hierarchy，輸出結果如output。

Report Configuration

uvm_top 可以使用set_report_verbosity_level_hier()這個function設定global report的詳細程度，reporting mechanism會影響simulation中所有component report的詳細程度。
